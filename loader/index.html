NumBry tests

Wasthon runs real CPython C extensions — and the numpy / pandas / scipy scientific stack — in the browser on top of Brython. Each dashboard below runs an upstream project's own test suite against the wasm builds, one fresh page per module, with per-module scores.

Test-suite dashboards

  • ▶ CPython suite dashboard — CPython's own Lib/test/test_*.py against the 25 ported modules (the authoritative behavioural reference).
  • ▶ numpy suite dashboard — modules from numpy 2.5.1's own suite (numpy/**/tests) against the C core + numpy.random wasm. Single module: test-numpy.html?test=….
  • ▶ scipy.ndimage suite dashboard — SciPy 1.14.1's own scipy/ndimage/tests against the compiled _nd_image/_ni_label wasm. Single module: test-scipy.html?test=….
  • ▶ scipy.special suite dashboard — SciPy 1.14.1's own scipy/special/tests against the nine Fortran-free extensions (cephes/xsf/Boost.Math) wasm, dstevr on lapack_lite. Single module: test-scipy-special.html?test=….
  • ▶ pandas suite dashboard — pandas 2.2.3's own pandas/tests/tslibs + libs against the 43 compiled pandas._libs wasm modules. Single module: test-pandas-mod.html?test=….

Smoke pages

  • ▶ numpy — arrays, reductions, ufuncs, @, slicing, subclassing, linalg.
  • ▶ pandas — Series/DataFrame/groupby/merge/pivot/read_csv/to_json… (20 checks).
  • ▶ scipy.ndimage — filters, interpolation, morphology, label/measurements (19 checks).
  • ▶ matplotlib — the Agg C++ renderer + FreeType + kiwisolver in wasm: draws a real figure and blits buffer_rgba() onto a canvas.
  • ▶ seaborn — the full stack in ONE wasm (numpy + lapack_lite + pandas + matplotlib): lineplot/histplot/kdeplot/regplot with real np.linalg.
