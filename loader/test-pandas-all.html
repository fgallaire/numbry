wasthon — pandas suite dashboard

Runs pandas 2.2.3's own tslibs + libs C-extension test modules against the combined numpy + numpy.random + pandas._libs wasm. brython + the VFS blobs + the wasm load once and the modules run sequentially in one interpreter — no per-module ~50 MB reload. Per-module detail (failures, timings) goes to the browser console; the page shows the table. Cross-module state is shared, so a module polluting the interpreter can affect later ones.
