wasthon — pandas suite dashboard

Runs modules from pandas 2.2.3's own test suite (the pandas/tests/tslibs + pandas/tests/libs C-extension-facing subset) against wasthon — the 43 compiled pandas._libs wasm modules — each in a fresh page (test-pandas-mod.html?test=…) so no module contaminates another. @given (hypothesis) tests are skipped. A minimal pytest shim supplies raises/mark.parametrize/ fixture/… Scores are numpy's own assertions. Only the pytest-portable subset is listed — modules needing numpy.random, C test-extensions, ctypes/mmap/zoneinfo, or hypothesis are out of scope. Failures are real bridge/behaviour gaps (the backlog to fix); a few come from the shim's fixture limits.
